PHASE 3 · 에이전트 심화
👋 오늘의 한마디

AI 에이전트는 스스로 생각하고, 스스로 행동하고, 스스로 판단합니다. 그 핵심이 바로 "도구 사용(Tool Use)"입니다. 검색, 계산, API 호출 같은 외부 도구를 동적으로 선택해 사용함으로써, 에이전트는 단순한 언어 모델을 벗어나 실제 문제를 푸는 행동 주체가 됩니다. 오늘은 ReAct, Plan-and-Execute, LATS 같은 설계 패턴을 통해 도구 사용의 메커니즘을 배웁니다.

🎯 오늘의 팁 2선
#133 📊 중급 도구 사용

Tool Use: 검색·계산·API 호출 연동

에이전트가 "오늘 환율이 뭔가요?"라는 질문에 정답을 주려면, 자신의 학습 데이터에만 의존해서는 불가능합니다. 실시간 검색 도구를 호출해야 합니다. 이렇게 외부 도구를 상황에 맞게 선택해 사용하는 것이 Tool Use입니다.

도구 사용의 3가지 핵심 패턴: 1) 검색(Search) - 웹에서 정보 조회, 2) 계산(Calculation) - 복잡한 수식 처리, 3) API 호출(API Call) - 외부 서비스 연동. 에이전트는 이 도구들을 필요에 따라 조합하며 사용합니다.

💬 실제 동작 흐름
Q: "서울에서 뉴욕까지 항공료는?" ↓ A (에이전트 판단): "이건 실시간 가격 정보가 필요하네" ↓ [Tool: Search API 호출] ↓ 조회 결과: 항공료 데이터 반환 ↓ A (최종 답변): "현재 항공료는 USD 950입니다"

핵심: 에이전트는 단순히 도구를 호출하는 것이 아니라, 언제 어떤 도구를 써야 할지를 판단하고 결과를 해석해 다음 단계를 결정합니다.

#134 🚀 고급 설계 패턴

에이전트 루프 설계 패턴: ReAct·Plan-and-Execute·LATS

Tool Use를 효과적으로 구현하기 위해 개발자들이 만든 세 가지 핵심 패턴이 있습니다:

① ReAct (Reasoning + Acting)
각 단계마다 "왜 이 도구를 쓸까?" → "도구 호출" → "결과 해석"을 반복합니다. 투명성이 높고 디버깅이 쉬운 장점이 있습니다.

② Plan-and-Execute
먼저 전체 계획을 세운 후(Plan), 각 단계를 순차적으로 실행(Execute)합니다. 대규모 작업에 효율적입니다.

③ LATS (Language Agent Tree Search)
여러 경로를 동시에 탐색하고 각 경로의 성공 확률을 평가해 최적의 루트를 선택합니다. 복잡한 문제 해결에 강합니다.

💬 세 패턴의 비교
ReAct: "시장 분석 필요 → Search API 호출 → 결과 확인 → 다음 도구 선택" → 한 단계씩 투명하게 진행 Plan-and-Execute: "1단계: 시장 조사, 2단계: 경쟁사 분석, 3단계: 전략 수립" → 전체 순서 결정 후 실행 LATS: "경로A: 이렇게 해볼까? 성공 가능성 60% 경로B: 이렇게는? 성공 가능성 85% → 경로B 선택 후 실행"

선택 기준: 작은 문제는 ReAct, 대규모 프로젝트는 Plan-and-Execute, 복잡한 의사결정은 LATS를 사용하세요.

💡 오늘의 인사이트

"도구 사용"은 에이전트가 AI를 넘어 "자율 행동 주체"가 되는 분기점입니다. 그동안 배운 프롬프팅, 체인 구축, 메모리 설계는 모두 "어떻게 생각할까"에 관한 것이었습니다. 하지만 Tool Use부터는 "어떻게 행동할까"의 영역으로 진입합니다. ReAct로 기본을 다지고, Plan-and-Execute로 확장하고, LATS로 최적화하는 이 과정 속에서 여러분의 에이전트는 진짜 문제 해결 능력을 갖추게 됩니다.

📖 오늘의 용어
Tool Use
Tool Use / Function Calling
AI 에이전트가 필요에 따라 검색, 계산, API 등 외부 도구를 동적으로 호출하고 그 결과를 해석해 다음 행동을 결정하는 능력. 에이전트가 외부 세계와 상호작용하는 핵심 메커니즘입니다.
ReAct 패턴
ReAct (Reasoning + Acting)
에이전트가 "생각(Reasoning)"과 "행동(Acting)"을 번갈아 반복하는 설계 패턴. 각 단계에서 왜 이 도구를 쓰는지, 결과가 맞는지를 명시적으로 검증하므로 투명성과 신뢰도가 높습니다.
