#131 🌱 입문 에이전트 정의

AI 에이전트란 무엇인가
챗봇 vs 에이전트의 차이

AI 에이전트(Agent)는 주어진 목표를 달성하기 위해 스스로 계획을 세우고, 실행하고, 결과를 평가하며 반복하는 자율적인 AI 시스템입니다. 단순히 사용자의 질문에 답변하는 것을 넘어, 복잡한 작업을 여러 단계에 걸쳐 주도적으로 수행합니다.

챗봇은 입력→처리→출력의 일방향 구조입니다. 사용자가 묻고, AI가 답합니다. 하지만 에이전트는 순환 구조로 작동합니다. 목표를 인지한 후, 필요한 도구를 선택하고, 실행하고, 그 결과를 평가해서 다음 단계를 결정합니다.

💬 예시로 이해하기
📞 챗봇 vs 에이전트 [챗봇의 방식] 사용자: "내일 서울에서 부산 출장이야. 뭘 챙겨야 해?" 챗봇: "기차표, 숙소 예약, 날씨 확인, 출장용 서류 등을 챙기세요." → 끝. 나머지는 사용자가 하나하나 [에이전트의 방식] 사용자: "내일 서울→부산 출장 준비해." 에이전트: 1️⃣ (인지) "서울→부산, 명일 출발이군" 2️⃣ (계획) "기차 예약 → 숙소 확인 → 날씨 체크 → 짐 리스트 작성" 3️⃣ (실행) - 네이버에서 기차표 검색 후 예약 - 야놀자에서 숙소 확인 및 예약 - 기상청에서 내일 날씨 조회 - 출장 필수 짐 리스트 자동 생성 4️⃣ (기억) 예약번호, 숙소주소, 날씨정보 저장 다음 질문에서 참고 → 사용자가 할 일 없음. 에이전트가 다 함

#132 📚 중급 아키텍처 기초

에이전트의 4대 구성요소
인지·계획·실행·기억

모든 에이전트는 네 가지 핵심 모듈로 이루어져 있습니다. 각 모듈이 유기적으로 연동되어야만 에이전트가 제대로 작동합니다.

1. 인지(Perception)
사용자의 요청과 환경 정보를 해석합니다. "뭘 해야 하는가"를 파악하는 단계입니다. 자연어 처리, 문맥 이해, 데이터 수집이 여기에 해당합니다.

2. 계획(Planning)
목표 달성을 위한 단계별 전략을 수립합니다. 어떤 도구를 사용할지, 어떤 순서로 진행할지를 결정합니다. 이를 "액션 플랜(Action Plan)"이라 부르기도 합니다.

3. 실행(Action)
계획에 따라 실제로 작업을 수행합니다. 외부 API 호출, 데이터베이스 접근, 파일 생성 등 구체적인 행동을 취합니다.

4. 기억(Memory)
과거 상호작용을 저장하고 학습합니다. 이전 대화의 맥락, 실행 결과, 사용자의 선호도를 기억하여 다음 요청에 더 정확하게 대응합니다.

💬 4대 요소가 작동하는 구체적 흐름
시나리오: "이번 주 화요일 10시에 팀장과 1:1 미팅 잡아줘" [1단계] 인지(Perception) 입력: "이번 주 화요일 10시에 팀장과 1:1 미팅 잡아줘" 파싱: 날짜(화요일), 시간(10:00), 대상(팀장), 작업(미팅 예약) [2단계] 계획(Planning) - 현재 캘린더 조회 - 팀장의 캘린더 접근 권한 확인 - 10시 시간대 충돌 체크 - 필요 시 시간 재조정 [3단계] 실행(Action) - Google Calendar API 호출 - 미팅 일정 생성 - 팀장에게 초대장 발송 - Slack 알림 설정 [4단계] 기억(Memory) 저장할 정보: - 사용자는 팀장과 주 1회 미팅 선호 - 보통 화요일 10시 고정 선호 - 1:1 미팅 시간은 평균 30분 → 다음 요청 때 이 정보들을 활용해서 더 빠르고 정확하게 대응

💡 오늘의 인사이트

에이전트가 단순한 챗봇과 다른 이유는 "자율성"입니다. 챗봇은 대기하는 수동적 존재지만, 에이전트는 목표를 향해 능동적으로 움직입니다. 앞으로의 25주 과정에서 우리는 이 네 가지 요소를 각각 어떻게 설계하고, 최적화하며, 실전에서 조합하는지를 깊이 있게 배우게 됩니다. 오늘의 이 4가지 개념이 이후 모든 학습의 기초가 된다는 점을 명심하세요.
