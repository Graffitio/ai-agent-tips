#121 MON 🌱 입문 채용 시스템

직무기술서(JD) 작성 도우미

AI는 단순한 JD 템플릿 제공자가 아닙니다. 귀사의 문화, 기술 스택, 기대 역할을 입력하면 그 조직에 딱 맞는 생생한 JD를 생성합니다. 모호한 직무 설명 대신, 지원자가 명확하게 이해할 수 있는 수준의 구체성이 가능합니다.

핵심: 역할 배경, 핵심 책임 3~4개, 필수/우대 자격 이 세 가지를 제공하면 완성도 높은 JD가 나옵니다.

💬 프롬프트 예시
우리는 D2C 팻테크 스타트업이고, 마케팅팀에 수급담당자를 채용해. 역할 배경: - 현재 팀: 마케팅 PM 2명, 그래픽 디자이너 1명 - 우리의 기술 스택: 데이터 분석은 GA4, CRM은 Klaviyo - 성장 목표: 월 30% 유저 증가 이 조건으로 "Product Marketing Manager (수급)" JD를 작성해줘. - 핵심 책임 4개 - 필수 자격 5개 - 우대 사항 3개

#122 MON 📘 중급 채용 시스템

이력서 스크리닝 기준 설계

JD만 있어서는 부족합니다. 그 JD에 정량적 스크리닝 기준을 AI와 함께 설계해야 합니다. "경험 많은" 사람을 찾는 것이 아니라, "우리 조직의 성장에 즉시 기여할 사람"을 찾는 기준입니다.

스크리닝 기준은 3 레이어로 구성: 필수(MUST), 강력 추천(STRONG), 우대(NICE)

💬 프롬프트 예시
위의 JD를 바탕으로, 이력서 스크리닝 매트릭스를 만들어줘. 형식: - MUST HAVE (이 기준 불만족 = 탈락) - STRONG (이 기준 충족 = 우선 면접) - NICE TO HAVE (충족하면 가산점) 각 기준마다 이력서에서 확인할 구체적인 증거를 제시해줘. 예) "구글 애널리틱스 경험" → 이력서에서 찾을 증거: GA4 구현 프로젝트, 대시보드 설계 등

#123 TUE 📘 중급 채용 면접

면접 질문 세트 자동 설계

좋은 면접 질문은 후보자의 역량을 측정하는 정밀한 도구입니다. "강점이 뭐예요?"라는 일반적인 질문 대신, 상황 기반(Behavioral) 질문을 AI가 자동으로 생성할 수 있습니다.

각 질문마다 원하는 답변의 핵심 요소와 평가 기준까지 함께 얻을 수 있습니다.

💬 프롬프트 예시
위의 JD의 "Product Marketing Manager" 직급 면접을 준비해. 아래 핵심 역량 각각에 대해: 1) Behavioral 질문 1개 2) 좋은 답변의 핵심 요소 3가지 3) 평가 기준 (1~5점) 역량: - 데이터 기반 의사결정 능력 - 마케팅과 개발 간 협력 경험 - 스타트업 환경에서의 속도감 질문 형식은 "과거에 겪은 구체적인 상황 → 어떻게 했나 → 결과가 뭔가" 순으로

#124 TUE 🚀 고급 온보딩

온보딩 콘텐츠 자동 생성

신입 입사자가 첫 주에 "조직과 시스템에 대해 빠르게 이해"하도록 돕는 일은 꽤 번거롭습니다. AI는 조직 구조, 기술 스택, 문화 요소를 입력받아 체계적인 온보딩 로드맵을 생성합니다.

Day 1부터 Day 30까지 구체적인 마일스톤과 학습 리소스가 담긴 문서가 즉시 생성됩니다.

💬 프롬프트 예시
신입 Product Marketing Manager를 위한 30일 온보딩 플랜을 만들어줘. 회사 정보: - 팀 규모: 마케팅 3명, 개발 5명, 디자인 1명 - 기술 스택: React, Node.js, Klaviyo, GA4 - 조직 특성: 빠른 의사결정, 월간 스프린트 형식: - Week 1: 조직 이해 (팀 미팅, 문화 이해) - Week 2~3: 기술/도구 학습 (GA4, Klaviyo 심화) - Week 4: 첫 프로젝트 시작 - 각 day마다 구체적인 액션 아이템과 담당자 표시

#125 WED 🚀 고급 성과 관리

실무 적용 성과 측정 프레임워크

"이 프로젝트가 성공이었나?"를 판단하는 명확한 기준이 없으면, 조직은 뱅뱅 도돕니다. AI는 프로젝트의 목표, 제약, 타이밍을 입력받아 정량적 성공 지표(KPI)를 자동 설계합니다.

단순 숫자가 아닌, 비즈니스 가치로 직결되는 지표를 생성합니다.

💬 프롬프트 예시
Q: 우리는 신규 고객을 위한 온보딩 이메일 시퀀스(5개) 프로젝트를 막 시작했어. 프로젝트 정보: - 목표: 신규 고객의 Day 7 활성화율 25% → 40% 달성 - 예상 비용: 마케팅 수급담당자 시간 80h - 런칭 타이밍: 2주 후 - 런칭 후 측정 기간: 3개월 성공 지표 프레임워크를 만들어줘. 형식: Primary KPI(필수 달성), Secondary KPI(모니터링), Guardrail(피해야 할 것) 각각 3개씩 각 지표마다 타겟값, 측정 방법, 영향 해석까지

#126 WED 📘 중급 프로젝트 관리

프로젝트 킥오프 문서 자동 생성

프로젝트 시작 전, 팀이 같은 페이지에 있는지 확인하는 것은 필수입니다. AI는 프로젝트 개요, 목표, 타임라인, 리스크를 포함한 완성도 높은 킥오프 문서를 자동으로 만들어냅니다.

문서 작성에 걸릴 시간을 줄이고, 실제 토론에 집중할 수 있게 합니다.

💬 프롬프트 예시
프로젝트 킥오프 문서를 작성해줘. 프로젝트명: 신규 온보딩 이메일 시퀀스 구축 주요 담당자: - PM: 김철수 (전체 조율) - 마케팅 수급담당자: 이영희 (콘텐츠 설계) - 개발자: 박민준 (자동화) 목표: 신규 고객 Day 7 활성화율 25% → 40% 예상 기간: 6주 (설계 2주, 개발 3주, QA 1주) 예상 비용: 마케팅 수급담당자 80h, 개발 60h 문서 구성: 1. Executive Summary (1단락) 2. 프로젝트 목표 및 배경 3. 스코프 (포함/제외 항목) 4. 타임라인 (주간별 마일스톤) 5. 리스크 및 대응 방안 6. 성공 기준 7. 의존성 및 선행 조건

THU 리스크 관리 · 스프린트 회고 분석
#127 THU 📘 중급 리스크 관리

리스크 레지스터 자동 업데이트

프로젝트 초기에 리스크를 예측하는 것은 중요하지만, 진행 중 상황 변화를 반영하는 것이 더 중요합니다. AI는 주간 스프린트 결과를 입력받아 새로운 리스크 항목을 자동으로 제안하고 기존 리스크의 심각도를 업데이트합니다.

리스크를 식별하고 → 대응책을 설계하는 전체 사이클을 가속화합니다.

💬 프롬프트 예시
온보딩 이메일 프로젝트의 리스크 레지스터를 업데이트해줘. 이번 주(Week 3) 스프린트 결과: - 계획: 이메일 템플릿 5개 완성 - 실제: 템플릿 3개만 완성 - 이유: 디자인 시스템 조정으로 재작업 필요 - 추가 발견: Klaviyo 커스텀 필드에 대한 개발팀의 학습곡선 가팜 현재 리스크 레지스터: - 데이터 품질 문제 (High) - 일정 지연 (Medium) - 스타트 데이터 부족 (Low) 추가될 새로운 리스크들을 제안하고, 각 리스크마다: - 영향도, 확률, 현재 상태 - 대응 전략 (Avoid/Mitigate/Transfer) - 담당자와 모니터링 방법

#128 THU 🚀 고급 팀 성장

스프린트 회고 분석·패턴 도출

스프린트 회고(Retrospective)는 중요하지만 종종 피상적입니다. AI는 회고 논의 내용을 입력받아 패턴을 자동으로 도출합니다. "왜 계속 같은 문제가 반복되나?" 같은 근본적인 질문에 답하게 됩니다.

다음 스프린트의 개선 액션도 자동으로 제안됩니다.

💬 프롬프트 예시
지난 4주간의 스프린트 회고 기록을 분석해줘. Week 1 회고: - 잘한 점: 일일 스탠드업이 효과적 - 개선점: 개발-디자인 간 의사소통 부족 - 액션: 매일 10:30 Quick Sync 추가 Week 2 회고: - 잘한 점: 개발-디자인 Quick Sync 효과 있음 - 개선점: PM이 요구사항을 너무 자주 바꿈 - 액션: 요구사항은 스프린트 시작 전에만 수정 Week 3 회고: - 잘한 점: 요구사항 안정화 - 개선점: Klaviyo 문서 부족으로 개발 막힘 - 액션: 외부 Klaviyo 전문가 상담 받기 Week 4 회고: - 잘한 점: 전문가 상담 유용함 - 개선점: QA 과정에서 많은 버그 발견 - 액션: 개발 중간에 QA 리뷰 추가 분석할 것: 1) 패턴: 같은 주제가 반복되는가? 2) 근본 원인: 이 패턴의 근본 원인은? 3) 체계적 개선안: 다음 4주를 위한 구조적 개선

#129 FRI 🚀 고급 의사결정

의사결정 매트릭스 자동 생성

좋은 조직은 의사결정 기준이 명확합니다. "어떤 기준으로 선택하나?"라는 질문에 즉시 답할 수 있어야 합니다. AI는 여러 선택지와 평가 기준을 입력받아 정량화된 의사결정 매트릭스를 생성합니다.

감정이 아닌 데이터 기반 의사결정이 가능해집니다.

💬 프롬프트 예시
다음 선택에 대한 의사결정 매트릭스를 만들어줘. 상황: 우리는 마케팅 자동화 플랫폼을 선택해야 해. 현재 Klaviyo 사용 중이지만, 다른 옵션들도 평가 중. 선택지: 1) Klaviyo (현재) - $300/month 2) ActiveCampaign - $200/month 3) Brevo (구 Sendinblue) - $100/month 4) HubSpot - $500/month 평가 기준 (가중치): - 가격 (20%) - 기능 완성도 (30%) - 고객 지원 (15%) - 통합 가능성 (25%) - 학습 곡선 (10%) 형식: 각 기준별 1~5점 평가 + 가중치 계산 + 최종 스코어

#130 FRI 🚀 고급 학습 종합

Phase 2 총정리: 실무 적용 마스터

Phase 1에서 배운 프롬프트 기본기가 Phase 2에서 HR, PM, 운영 전체 영역에 어떻게 확산되는지 총정리합니다. 개별 팁들이 아닌, 조직 시스템 전체에서 AI를 엮는 방법을 배웁니다.

다음 Phase 3(리더십·전략)의 진입을 위한 확실한 기반을 마련합니다.

💬 종합 점검 리스트
Phase 2를 마치며 점검할 것: ✅ 채용 루프 (MON-TUE) - JD 작성 → 스크리닝 기준 → 면접 질문 → 온보딩까지 한 번에 준비할 수 있는가? ✅ 프로젝트 루프 (WED) - 킥오프 문서 → 성공 기준 → 측정 프레임워크 세 가지를 한 패키지로 만들 수 있는가? ✅ 운영 루프 (THU-FRI) - 주간 스프린트 결과 → 리스크 업데이트 → 회고 분석 → 의사결정까지 순환하는가? 이 세 루프가 조직 내에서 자동으로 돌아가도록 설계하는 것이 Phase 2의 목표입니다. 다음 Phase 3에서는: - 경영진 보고서 자동 작성 - 전략 문서 생성 - 조직 성과 관리 시스템 구축 ...을 배우게 됩니다.

💡 오늘의 인사이트: 큰 그림

지난 3개월은 "도구"를 배웠습니다. 이번 주는 "시스템"을 경험합니다.

개별 프롬프트들이 어떻게 조직 전체의 의사결정을 자동화하는지 봅시다. 직무기술서 하나를 만들 때 이미 채용 전체 프로세스가 설계됩니다. 프로젝트 킥오프할 때 이미 성공의 기준과 실패의 신호가 정해집니다. 스프린트 회고를 분석할 때 팀의 성장 경로가 보입니다.

이것이 AI를 "사용하는 것"에서 "시스템화하는 것"으로 나아가는 길입니다. Phase 3에서는 이 시스템을 조직 리더십 수준으로 확대합니다.

📖 이번 주의 핵심 용어
JD (Job Description)
직무기술서
직급, 역할, 책임, 필요 자격을 명확히 기술한 문서. AI와 함께 만들면 조직 특성이 담긴 구체적인 JD가 나옵니다.
Behavioral Interview
행동 기반 면접
"과거에 어떤 상황에서 어떻게 했나"를 묻는 면접 방식. 역량을 가장 정확하게 평가할 수 있습니다.
KPI (Key Performance Indicator)
핵심 성과 지표
프로젝트나 팀의 성공을 측정하는 정량적 지표. 비즈니스 목표와 직결되어야 합니다.
Risk Register
리스크 레지스터
