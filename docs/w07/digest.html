#061 🌱 입문 프롬프트 개선

Before/After로 보는 프롬프트 개선

같은 작업이라도 프롬프트를 어떻게 쓰느냐에 따라 결과가 180도 달라집니다. 핵심은 AI에게 "무엇을 해야 하는가"뿐만 아니라 "어떻게 생각해서 그렇게 해야 하는가"까지 알려주는 것입니다.

💬 Before / After
❌ Before: "고객 만족도 이메일 작성해줘" ✅ After: "우리 고객이 제품을 샀지만 아직 완전히 만족하지 못한 상태라고 가정해. 이메일의 목표는 불안감을 없애고 제품 가치를 다시 보여주는 것이야. 톤은 따뜻하지만 비즈니스스럽게. 3문단, 각 150자 이내로 작성해줘."

#062 📊 중급 실수 패턴

흔한 프롬프트 실수 Top 10

1. 방향성 없는 질문 ("좋은 콘텐츠 아이디어 줘" ❌)
2. 너무 긴 문맥 (AI가 초점을 잃음)
3. 모호한 기준 ("자연스럽게" vs "SEO 최적화" 명시 필요)
4. 역할 정의 부족 ("마케터처럼" 같은 페르소나 부재)
5. 제약 조건 생략 (글자 수, 형식, 타겟 미지정)
6. 예시 없음 (원하는 스타일을 직접 보여주지 않음)
7. 한 번에 여러 작업 (분석+추천+실행까지 한 프롬프트)
8. 결과만 원함 (과정과 근거 요청 없음)
9. 테스트 없이 대량 사용 (한 번 검증 후 스케일)
10. 피드백 미반영 (한 번의 답변으로 끝냄)

#063 📊 중급 업종별 최적화

업종별 프롬프트 베스트 프랙티스

각 업종은 고유한 프롬프트 패턴을 가집니다.

🏪 이커머스: 상품 특성 + 구매자 페인포인트 + 객관적 수치
📱 SaaS: 문제 정의 + 솔루션 + 메트릭
📰 미디어: 타겟 이해도 + 논거 + 출처 명시
🏢 B2B 영업: 구매자 이해 + 이해관계 + 예산 규모
👤 HR/교육: 학습 목표 + 수준 + 결과물 용도

#064 🚀 고급 테스트 방법론

프롬프트 A/B 테스트 방법론

대량 작업에 프롬프트를 쓸 때는 먼저 작은 샘플(5~10개)로 A/B 테스트를 합니다. 변수 하나만 바꾸고 결과를 비교한 뒤, 더 나은 버전으로 확대하세요.

💡 A/B 테스트 프레임
Step 1: 베이스 프롬프트 고정 (상황 + 역할 + 목표) Step 2: 변수 선택 (1가지만) - A버전: 구체성 높음 - B버전: 창의성 높음 Step 3: 샘플 5개로 각각 실행 Step 4: 검증 기준 정의 (길이, 톤, 정확도, 실용성 등) Step 5: 승자 선택 → 확대 적용

#065 🚀 고급 워크플로우

프롬프트에서 워크플로로 전환하기

"프롬프트"는 한 번의 질문이고, "워크플로"는 여러 단계의 작업 흐름입니다. 같은 일을 반복할 때는 워크플로로 만들어서 1) 입력 → 2) 처리 → 3) 검증 → 4) 출력 단계를 자동화하세요.

💡 워크플로 구성 예
🔄 뉴스레터 콘텐츠 자동화 1️⃣ 입력: CSV 파일로 주제 5개 제공 2️⃣ 처리: - 각 주제별 AI가 개요 작성 - 검토자 피드백 수집 - 최종 버전 작성 3️⃣ 검증: 길이/톤/링크 자동 체크 4️⃣ 출력: HTML로 변환 후 메일링 시스템 연동

#066 🌱 입문 이메일 자동화

이메일: 상황별 톤 자동 조절

같은 메시지도 누구에게, 어느 상황에서 보내는지에 따라 톤이 달라져야 합니다. 프롬프트에 "상황 변수"를 추가하면 AI가 자동으로 톤을 조절합니다.

💬 상황별 톤 조절 프롬프트
기본 정보: - 수신자: [이름] - 상황: [구매 후 1주일/구매 의도 미정/이탈 고객] - 목표: [구매 독려/불안감 해소/재관심 유도] 내용: "우리 [제품명]을 [상황]에 있는 분께 [목표]를 위해 이메일을 쓰겠습니다. 톤은 [친근함/전문성/아쉬움] 중 어느 쪽인가요?"

#067 📊 중급 보고서 작성

보고서: 데이터를 스토리로 변환

숫자만 나열하는 보고서는 읽지 않습니다. 데이터 뒤에 "왜" 그런 결과가 나왔는지 스토리를 담아야 합니다. AI에게 "분석 → 인사이트 → 액션"의 3단계로 나누어 요청하세요.

💡 보고서 구조 프롬프트
1️⃣ 분석: 데이터만 제시 "3월 신규 고객 유입 25% 증가" 2️⃣ 인사이트: 원인 찾기 "인스타그램 리타게팅 광고의 효과" 3️⃣ 액션: 다음 단계 "4월에도 같은 채널에 예산 25% 증액, 다른 콘텐츠 3가지 A/B 테스트"

#068 📊 중급 제안서 작성

제안서: 설득 구조 자동 설계

제안서를 쓸 때는 "상황 → 문제 → 우리 솔루션 → 결과 → CTA"의 5단계 설득 구조를 따릅니다. AI가 이 구조를 이해하도록 명확히 지시하면, 각 섹션이 논리적으로 연결됩니다.

💡 제안서 5단계 프롬프트
1️⃣ 상황: "귀사는 마케팅팀 규모가 3명인데..." 2️⃣ 문제: "따라서 감당할 수 없는 작업량으로..." 3️⃣ 솔루션: "우리의 AI 콘텐츠 플랫폼은..." 4️⃣ 결과: "이를 통해 월 30시간 절감," 5️⃣ CTA: "다음 주 데모를 신청하시면..."

#069 🚀 고급 법률 문서

계약서·법률 문서 초안 보조

AI로 법률 문서를 완성할 수는 없지만, 초안을 빠르게 만들고 변호사의 리뷰 시간을 줄일 수는 있습니다. 핵심은 "계약 조건을 명확히 입력"하고, 생성된 문서를 반드시 전문가 검토를 거치는 것입니다.

💡 법률 문서 입력 템플릿
- 계약 주체: [회사 A] ↔ [회사 B] - 계약 기간: [2026년 4월 1일 ~ 2027년 3월 31일] - 주요 의무: * A의 의무: [서비스 제공 및 SLA 99.9%] * B의 의무: [월 100만원 지급] - 특수 조건: [경쟁금지 1년, 비밀유지] - 종료 조건: [상호 동의 또는 90일 전 통보] "위 조건을 반영한 서비스 이용 약관 초안을 한국 법 기준으로 작성해줘. 변호사 리뷰 전 체크리스트도 함께."

#070 🚀 고급 다국어 문서

다국어 비즈니스 문서 동시 생성

한 번의 프롬프트로 여러 언어의 문서를 동시에 생성하되, 단순 번역이 아니라 "각 언어권의 비즈니스 관례"를 반영하도록 지시하세요.

💡 다국어 생성 프롬프트
다음 제안서를 한국어, 영어, 일본어로 동시에 작성해줘. 단, 각 버전은: 한국어: 존댓글, 존경-겸손한 톤 영어(US): 직설적, 가치 중심 영어(UK): 정중한, 상세한 설명 일본어: 존경어 사용, 상호 이익 강조 각 버전의 마지막에 "문화적 고려사항"을 3줄씩 덧붙여줘.

💡 이번 주의 핵심

Phase 2는 "일에 맞추는" 단계입니다. 기본 프롬프트를 알았으니, 이제는 그것을 자신의 업종, 팀 문화, 문서 유형에 맞게 변형하고 반복하고 시스템화하는 방법을 배웁니다. 이번 주 10가지 팁을 모두 실습하면, 당신은 더 이상 "AI 사용자"가 아니라 "AI를 일의 도구로 자유자재하게 다루는 사람"이 됩니다.
