👋 오늘의 한마디

고객 응대의 80%는 반복되는 질문입니다. "자주 묻는 질문을 얼마나 잘 자동화하느냐"에 따라 팀의 생산성과 고객 만족도가 결정됩니다. 오늘은 AI를 활용해 FAQ 챗봇을 설계하고, 들어오는 문의를 자동으로 분류·라우팅하는 두 가지 실무 기법을 배웁니다.

🎯 오늘의 팁 2선
#111 🌱 입문 FAQ 봇

FAQ 챗봇 프롬프트 설계 — 자주 묻는 질문 자동 응답

좋은 FAQ 챗봇은 "학습 데이터 + 응답 규칙 + 톤 앤 매너"의 조합입니다. 단순히 답변만 나열하는 것이 아니라, 고객의 질문 의도를 파악하고 그에 맞는 응답을 구조화해야 합니다.

핵심은 FAQ 데이터베이스를 프롬프트에 임베드하는 것입니다. "이런 질문이 들어오면 이렇게 답해"라는 패턴을 명확히 제시하면, AI는 새로운 질문이 들어올 때도 그 패턴을 따릅니다.

💬 프롬프트 구조
당신은 [회사명] 고객 지원 챗봇입니다. 【역할】 - 고객의 질문을 빠르고 정확하게 답변 - 답변할 수 없는 경우 인간 담당자로 연결 제안 - 항상 친절하고 전문적인 톤 유지 【FAQ 데이터베이스】 Q1: 배송은 며칠 걸리나요? A1: 일반배송은 2-3영업일, 특급배송은 당일 배송됩니다. Q2: 반품 기간은? A2: 상품 수령 후 7일 이내 반품 가능합니다. (미개봉, 손상 없는 경우) Q3: A/S 비용이 드나요? A3: 보증 기간 내 제품 결함으로 인한 A/S는 무료입니다. 【고객 질문】 {customer_question} 【응답 지침】 1. FAQ에서 유사한 답변이 있으면 그것을 바탕으로 응답 2. 정확한 정보가 없으면 "담당자 확인 후 연락드리겠습니다"라고 안내 3. 각 답변 마지막에 추가 도움이 필요한지 확인 질문 포함

#112 📚 중급 분류 자동화

고객 문의 분류·라우팅 자동화 — 문의 유형별 자동 배정

모든 문의를 한 명의 담당자에게 몰아주면 병목 현상이 생깁니다. "이 문의는 배송팀으로, 이것은 기술팀으로, 이것은 자동 응답으로" 라우팅하려면 먼저 정확하게 분류해야 합니다.

AI가 문의를 카테고리로 자동 분류하고, 각 카테고리마다 정해진 응답 또는 담당자로 연결하는 시스템을 만들면, 수동 작업 80%를 줄일 수 있습니다.

💬 분류·라우팅 프롬프트
당신은 고객 문의 분류 시스템입니다. 【분류 카테고리 정의】 1. 배송 관련 (Shipping) → 배송 기간, 배송비, 주소 변경 등 → 라우팅: logistics@company.com 2. 환불/반품 (Refund/Return) → 반품 신청, 환불 현황 등 → 라우팅: returns@company.com 3. 기술 지원 (Technical) → 상품 사용법, 오류, 호환성 등 → 라우팅: support@company.com 4. 자동 응답 (Auto-Response) → FAQ로 답변 가능한 일반 질문 → 라우팅: FAQ 챗봇 5. 에스컬레이션 (Escalation) → 특수 상황, 불만 사항 → 라우팅: manager@company.com 【문고 분석 프로세스】 1. 고객이 보낸 문의 텍스트를 읽음 2. 주요 키워드 추출 (배송, 반품, 작동 안됨 등) 3. 가장 적합한 카테고리 1개 선택 4. 라우팅 대상 명시 【출력 형식】 카테고리: [분류] 라우팅: [담당자/시스템] 우선순위: [높음/중간/낮음] 자동 응답 초안: [고객에게 먼저 보낼 안내 메시지]

💡 오늘의 인사이트

FAQ 자동화와 문의 라우팅은 단순히 "업무 효율화"가 아닙니다. 고객이 기다리는 시간을 줄이고, 담당자는 정말 필요한 일에만 집중하게 됩니다. 데이터를 모으고, 패턴을 정리하고, AI로 자동화하는 이 과정이 바로 팀의 생산성을 3배로 만드는 방법입니다. 이번 주말, 당신의 회사에서 가장 자주 받는 질문 20개를 모아 보세요. 다음 주는 그것을 FAQ 챗봇으로 만드는 방법을 배웁니다.

라우팅 (Routing)
Routing

프롬프트 임베딩 (Prompt Embedding)
Prompt Embedding
FAQ 데이터나 분류 규칙을 프롬프트에 직접 포함시키는 기법. 정보가 많을수록 AI의 응답이 정확해집니다.
