PHASE 1 · 프롬프트 기본기
👋 오늘의 한마디

AI의 가장 강력한 패턴 중 하나는 "자신의 출력을 다시 입력으로 사용하는 것"입니다. 이것이 바로 리커시브 프롬프트의 본질이죠. 그리고 이 패턴들을 팀 전체가 재사용할 수 있도록 정리하는 것이 프롬프트 라이브러리입니다. 오늘은 AI 워크플로우를 한 단계 업그레이드하는 두 가지를 배웁니다.

🎯 오늘의 팁 2선
#059 🚀 고급 재귀 패턴

고급 재귀 패턴 — 리커시브 프롬프트: 자기 참조 루프

재귀(Recursion)는 프로그래밍에서 함수가 자기 자신을 호출하는 기법입니다. AI 프롬프트에도 같은 원리를 적용할 수 있습니다. 즉, AI의 출력을 다시 같은 패턴의 입력으로 넣어서 점진적으로 결과를 개선하는 것이죠.

예를 들어 "아이디어 100개를 생성한 뒤, 그 중 가장 좋은 10개를 선별하고, 그 10개를 더 정교하게 발전시키는" 과정은 모두 같은 프롬프트 패턴을 반복 사용하는 재귀 구조입니다.

💬 재귀 패턴 예시
🔄 Step 1 (생성): 프롬프트: "SNS 마케팅 아이디어 30개를 스타트업 B2B SaaS 관점에서 생성해줘." → 아이디어 30개 출력 🔄 Step 2 (필터링): 프롬프트: "[Step 1의 결과를 붙임] 이 중에서 실행 가능성이 높은 아이디어 Top 5를 뽑고, 각각 이유를 설명해줘." → 5개로 압축 🔄 Step 3 (심화): 프롬프트: "[Step 2의 결과를 붙임] 이 5개를 각각 2주 실행 계획으로 구체화해줘. (담당자, 일정, 예상 효과 포함)" → 최종 실행 계획

핵심: 같은 구조의 프롬프트를 3~4 사이클 반복하면 처음 요청의 정확도와 깊이가 눈에 띄게 높아집니다. 이것이 바로 "자기 참조 루프"의 힘입니다.

#060 🚀 고급 라이브러리 관리

고급 라이브러리 — 프롬프트 라이브러리 구축법

재사용 가능한 프롬프트들은 팀 자산입니다. 하지만 무작정 모아두면 검색도 어렵고 버전 관리도 복잡해집니다. 효과적인 프롬프트 라이브러리를 만들려면 다음 네 가지가 필수입니다.

📋 프롬프트 라이브러리 구축 4단계
1️⃣ 카테고리 분류 - 콘텐츠 작성 (블로그, SNS, 이메일) - 분석 (경쟁사, 시장, 데이터) - 개발 (코드 리뷰, 버그 분석, 문서화) - 전략 (아이디어 생성, 의사결정) 2️⃣ 프롬프트 템플릿 표준화 [프롬프트명] | [목적] | [입력값 형식] | [출력값 형식] | [예시] 3️⃣ 버전 관리 + 성능 평가 - v1.0 (초안) → v1.1 (개선) → v2.0 - 각 버전마다 "효과도", "실행시간" 기록 4️⃣ 공유 및 피드백 루프 - Notion/Confluence 같은 팀 위키에 정리 - "사용자 평가" 섹션 추가 (피드백 수집) - 월 1회 개선 회의

추가 팁: 각 프롬프트마다 "언제 쓰면 좋은가"를 명시하세요. 예: "경쟁사 3~5개 분석할 때", "주간 회의 요약이 필요할 때" 등. 이렇게 하면 팀원들의 검색과 활용이 훨씬 쉬워집니다.

💡 오늘의 인사이트

리커시브 프롬프트와 프롬프트 라이브러리는 개인 작업 → 팀 시스템으로의 전환점입니다. 개인이 AI를 잘 쓰는 것도 중요하지만, 팀 전체가 같은 "좋은 프롬프트"를 공유하고 개선할 수 있는 문화가 있으면 AI 활용의 효율성은 배가됩니다. 오늘 배운 재귀 패턴 3~4개만 정리해서 팀과 공유해 보세요. 그것이 바로 팀의 AI 역량을 한 단계 올리는 시작입니다.

📖 오늘의 용어
리커시브 프롬프트
Recursive Prompt
AI의 이전 출력을 다시 새로운 프롬프트의 입력으로 사용하여 점진적으로 결과를 개선하는 기법. 같은 구조의 질문을 반복하되 내용을 심화시켜 최종 결과물의 정교함을 높입니다.
프롬프트 라이브러리
Prompt Library
팀이나 조직이 반복적으로 사용할 수 있는 검증된 프롬프트들을 카테고리별로 정리하고 관리하는 시스템. 버전 관리, 성능 평가, 사용 가이드를 포함하여 팀 자산으로서 지속적으로 개선됩니다.
