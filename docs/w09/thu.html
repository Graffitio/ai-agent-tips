👋 오늘의 한마디

데이터 분석의 첫 번째 장벽은 "엑셀 함수가 아니라 구조화"입니다. 오늘은 AI가 당신의 데이터를 어떻게 시각화하고 재구조화하는지, 그리고 왜 피벗 테이블이 없어도 크로스탭 분석이 가능한지를 배웁니다. 자연어 하나로 차트와 표를 만드는 경험이 여러분의 데이터 리터러시를 한 단계 끌어올릴 거예요.

🎯 오늘의 팁 2선
#087 🌱 입문 차트 생성

자연어로 차트 만들기

더 이상 "엑셀로 꺾은선 그래프를 그려야 한다"는 생각을 버리세요. AI에게 원하는 차트의 형태와 당신의 데이터를 구체적으로 설명하면, 바로 시각화 코드나 데이터 포맷을 제시합니다.

핵심은 "무엇을 X축에, 무엇을 Y축에 놓을지"를 명확히 하는 것입니다. 월별 매출 추이라면 "월은 가로, 매출액은 세로"라고 정확히 지시해야 AI가 올바른 차트를 생성합니다.

💬 데이터 시각화 프롬프트
📊 프롬프트: "우리 회사의 월별 매출 데이터가 있어. 2024년 1월부터 12월까지 매달 매출액(백만원)이 이렇게 되어. 1월: 45, 2월: 52, 3월: 48, 4월: 61, 5월: 58, 6월: 72, 7월: 89, 8월: 95, 9월: 82, 10월: 76, 11월: 88, 12월: 102 이 데이터를 꺾은선 그래프로 보여줄 수 있는 Python 코드를 작성해줘. X축은 월, Y축은 매출액으로."

#088 📊 중급 크로스탭 분석

피벗 테이블 없이 크로스탭 분석

크로스탭(교차표)은 행과 열이 만나는 지점에 값을 채우는 표입니다. 예를 들어 "부서별 · 지역별 매출"을 한눈에 보려면 피벗 테이블을 써야 할 것 같지만, AI에게 원본 데이터와 원하는 행·열을 알려주면 즉시 재구조화된 표를 만들어줍니다.

AI의 데이터 변형 능력은 엑셀의 자동화보다 훨씬 빠르고 유연합니다. 복잡한 함수를 짤 필요 없이 자연어로 의도만 전달하면 됩니다.

💬 크로스탭 생성 프롬프트
📋 프롬프트: "다음 판매 데이터를 크로스탭으로 정리해줄래? 부서,지역,Q1,Q2,Q3,Q4 영업팀,서울,120,145,168,192 영업팀,부산,85,92,105,118 마케팅팀,서울,65,78,82,95 마케팅팀,부산,45,52,58,68 ↓ 행(세로) = 부서 / 열(가로) = 지역 으로 각 셀에는 Q1+Q2+Q3+Q4 합계를 넣은 크로스탭 표를 만들어줘."

💡 오늘의 인사이트

데이터 분석에서 "기술"이 차지하는 비중은 생각보다 작습니다. 더 중요한 것은 "당신의 데이터가 무엇을 말하고 싶은가"를 이해하고, 그것을 AI에게 명확하게 전달하는 능력입니다. 오늘 배운 두 가지 — 자연어 차트 생성과 AI 크로스탭 분석 — 은 단순한 기술 팁이 아니라, 데이터와 대화하는 새로운 방식입니다. 이 능력이 쌓이면, 당신은 더 이상 "엑셀이 안 된다"고 느낄 일이 없을 겁니다.

📖 오늘의 용어
크로스탭
Cross-tabulation / Crosstab
두 개 이상의 범주형 변수의 교차 관계를 표로 나타낸 것. 행과 열의 교점에 빈도나 합계 등의 값을 배치합니다. 엑셀의 피벗 테이블과 동일한 개념입니다.
데이터 재구조화
Data Reshaping / Data Transformation
원본 데이터의 행과 열 배치를 변경하여 새로운 구조로 만드는 과정. AI를 활용하면 복잡한 함수 없이 자연어만으로 원하는 형태로 변환할 수 있습니다.
