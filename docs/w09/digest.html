#081 🌱 입문 기술 문서

기술 문서를 쉬운 말로 바꾸기

개발자들이 쓴 API 문서나 기술 스펙은 낯설고 딱딱합니다. 이 문서를 비개발자도 이해할 수 있는 언어로 번역하는 것이 첫 번째 실무 활용입니다.

핵심: "다음 기술 문서를 [대상 직군]도 이해할 수 있게 쉽게 설명해줘. 예시도 넣고, 도표로 정리해줄 수 있으면 더 좋아."

💬 활용 예시
📝 Prompt: "다음 REST API 문서를 마케팅팀도 이해할 수 있게 다시 써줘. 각 엔드포인트가 실제로 뭘 하는지, 어디서 쓰이는지 예시와 함께 설명해줘. [API 문서 원문 붙여넣기]" 📊 Output: - 각 API의 역할을 마케팅 용어로 설명 - 실제 마케팅 캠페인에서 쓰는 사례 - 일반인도 알 수 있는 다이어그램

#082 🔥 고급 문서 자동화

API 문서·가이드 자동 생성

API 코드만으로도 완성도 높은 API 문서를 자동 생성할 수 있습니다. AI가 코드를 읽고 사용법, 에러 처리, 예제까지 정리하는 것이죠.

개발자: "이 코드의 API 문서를 OpenAPI/Swagger 형식으로 생성해줘. 각 엔드포인트마다 사용 예시, 에러 코드, 권한 요구사항을 포함해서."

💬 활용 예시
📝 Prompt: "다음은 우리 백엔드 코드야. 이를 바탕으로 완전한 API 문서를 OpenAPI 3.0 형식으로 만들어줘. 1. 모든 엔드포인트 나열 2. 요청/응답 스키마 3. 실제 cURL 사용 예시 4. 각 HTTP 상태 코드별 에러 설명 [Python/Node.js 코드 붙여넣기]" 📊 Output: - OpenAPI YAML 파일 생성 - 각 엔드포인트별 사용 가이드 - 클라이언트가 바로 쓸 수 있는 예제

#083 ⚡ 중급 지식 관리

사내 위키·FAQ 자동 정리

흩어진 문서들을 하나의 체계적인 위키로 통합하고, 자주 묻는 질문을 자동으로 추출해 FAQ를 만듭니다.

프롬프트: "다음 사내 문서들을 읽고, 핵심 정보를 주제별로 분류해서 위키 형식으로 정리해줘. 그리고 자주 나올 법한 질문 Top 10을 뽑아줄 수 있어?"

💬 활용 예시
📝 Prompt: "우리 팀의 온보딩 가이드, 프로젝트 문서, Slack 메시지들이 여기 있어. 이걸 다음 형식으로 정리해줄 수 있어? 1. 주제별 분류 위키 (마크다운) 2. 신입도 물어볼 법한 FAQ 20개 3. 각 문서의 최신성 체크리스트 [여러 문서 붙여넣기]" 📊 Output: - 카테고리별 정리된 위키 문서 - 검색 가능한 FAQ 데이터베이스 - 업데이트 필요한 항목 플래그

#084 🔥 고급 자동화

릴리스 노트 자동 생성 파이프라인

Git 커밋, GitHub Issues, 코드 변경사항을 자동으로 수집해서 전문가적인 릴리스 노트를 생성합니다. 여러 단계를 연결한 실전 워크플로우죠.

프롬프트 연쇄: ① 커밋 로그 요약 → ② 기능/버그 분류 → ③ 사용자 입장의 릴리스 노트 작성 → ④ 마케팅팀용 간략본 생성

💬 활용 예시
🔄 4단계 파이프라인: 1️⃣ (AI에게 커밋 로그 분석 요청) "v3.2.0 릴리스를 위한 지난 2주 커밋을 요약해줄 수 있어? 각 커밋의 의도와 영향 범위를 정리해줘." 2️⃣ (분류) "그 커밋들을 Feature(새기능), Bugfix(버그), Perf(성능)로 분류하고, 사용자가 알아야 할 항목만 골라줘." 3️⃣ (릴리스 노트 작성) "2️⃣ 결과를 바탕으로 개발자용 릴리스 노트를 마크다운으로 작성해줘. 마이그레이션 가이드도 포함해." 4️⃣ (마케팅 버전) "3️⃣ 릴리스 노트를 마케팅팀이 블로그에 쓸 수 있게 평이한 말로 다시 써줄 수 있어?" 📊 최종 Output: - 개발자용 상세 릴리스 노트 - 마케팅용 블로그 포스트 - 고객 공지용 요약본

#085 🔥 고급 지식 시스템

지식 그래프 기반 문서 연결

산재된 문서들 사이의 관계를 AI가 자동으로 발견해서 "이 문서도 읽으면 도움이 될 거야"라고 추천합니다. 지식 네트워크를 구축하는 고급 기법이에요.

프롬프트: "다음 문서들을 읽고, 문서 간 연관성을 분석해줄 수 있어? 각 문서가 어떤 개념을 다루고, 어느 문서와 연결되는지 JSON 그래프 형식으로 만들어줘."

💬 활용 예시
📝 Prompt: "우리 팀 위키의 이 50개 문서를 읽고 문서 간 연결 관계를 분석해줄 수 있어? Output 형식: { "nodes": [ {"id": "문서명", "category": "분류"} ], "links": [ { "source": "문서A", "target": "문서B", "relation": "관련개념/선행학습/심화내용" } ] } [50개 문서 텍스트]" 📊 Output: - 문서 간 연관성 JSON 그래프 - 각 문서의 '다음 추천 읽을거리' 자동 생성 - 학습 경로 자동 제안

#086 🌱 입문 데이터 분석

엑셀 데이터를 AI에게 분석시키기

엑셀 데이터를 AI에게 넣으면 추세, 이상치, 패턴을 자동으로 발견합니다. 통계 지식이 없어도 AI가 해석해줘요.

프롬프트: "다음 데이터를 분석해줄 수 있어? 어떤 추세가 보이고, 뭐가 이상한지, 어떤 인사이트가 숨어 있는지 알려줘."

💬 활용 예시
📝 Prompt: "월별 판매량, 고객수, 환불율 데이터가 여기 있어. 이를 분석해서 다음을 알려줄 수 있어? 1. 3개월 추세 (증가/감소/정체) 2. 이상한 달이 있는지 3. 변수들 간의 상관관계 4. 다음 분기 예상 [엑셀 데이터 또는 CSV 붙여넣기]" 📊 Output: - 각 변수별 트렌드 분석 - 이상 구간 플래그 - 원인 가설 제시 - 간단한 예측치

#087 🌱 입문 시각화

자연어로 차트 만들기

"이 데이터를 이렇게 보고 싶어"라고 말하면, AI가 Python/R/JavaScript 차트 코드를 생성합니다. 엑셀 차트 수준을 벗어나는 것이 가능해요.

프롬프트: "이 데이터로 [차트 유형] 차트를 만들어줄 수 있어? [도구] 코드로 부탁해. 대시보드에 쓸 거라서 보기 좋게 스타일링도 해줄 수 있으면 좋겠어."

💬 활용 예시
📝 Prompt: "월별 매출, 비용, 이윤 데이터가 있어. 1. 세 개를 한 차트에 보여줄 수 있을까? (매출/비용은 막대, 이윤율은 선) 2. Python plotly 코드로 부탁해 3. 대시보드 느낌으로 색상도 해줘 [CSV 데이터]" 📊 Output: - 완성도 높은 Plotly Python 코드 - 대시보드 스타일 적용 - 대화형 차트 (마우스 호버 등)

#088 ⚡ 중급 분석

피벗 테이블 없이 크로스탭 분석

엑셀의 피벗 테이블 없이, AI가 자동으로 데이터를 다차원으로 집계하고 분석합니다. "고객 나이별, 상품 카테고리별 매출"처럼 복합 분석도 한 번에요.

프롬프트: "이 데이터를 [행], [열] 기준으로 [값] 을 집계해줄 수 있어? 피벗 테이블 형식으로 정리하고, 각 셀의 의미도 설명해줘."

💬 활용 예시
📝 Prompt: "고객 데이터가 있어: - 나이대 (20s, 30s, 40s+) - 성별 (M/F) - 매출 - 반복 구매율 이 데이터를 나이대(행) x 성별(열)로 매출을 합산한 크로스탭으로 만들어줄 수 있어? 그 다음 반복 구매율도 같은 방식으로. [고객 데이터 CSV]" 📊 Output: - 나이대×성별 매출 크로스탭 - 나이대×성별 반복 구매율 크로스탭 - 각 행/열의 합계 및 증감률

#089 ⚡ 중급 데이터 정제

이상치 탐지와 데이터 클리닝

AI가 데이터의 오류, 이상치, 누락을 자동으로 찾아내고, 정제 방법을 제안합니다. 분석 전 데이터 품질을 확보하는 것이 핵심이에요.

프롬프트: "이 데이터에서 이상치나 오류가 있는지 찾아줄 수 있어? 각 항목별로 문제점과 해결 방안도 제시해줄 수 있으면 좋겠어."

💬 활용 예시
📝 Prompt: "2년치 거래 데이터가 여기 있어. 데이터 품질을 체크해줄 수 있어? 1. 누락된 값이 몇 개? 2. 말이 안 되는 값들 (예: 가격이 음수, 나이가 200살) 3. 형식 오류 (날짜 형식 불일치 등) 4. 각 문제별 정제 방법 제시 [CSV 데이터]" 📊 Output: - 이상치 목록 (행 번호 포함) - 누락 패턴 분석 - 정제 전/후 비교 - 정제된 CSV

#090 🔥 고급 통계

통계 분석 해석을 AI에게 맡기기

p-값, 신뢰도, 회귀 계수 같은 통계 용어를 AI가 평이하게 해석해줍니다. 통계 지식이 없는 의사결정자도 결과를 이해할 수 있어요.

프롬프트: "[통계 분석 결과]를 비통계 전공자도 이해할 수 있게 해석해줄 수 있어? 핵심 결론과 의사결정에 미치는 영향도 알려줄 수 있으면 좋겠어."

💬 활용 예시
📝 Prompt: "다음은 A/B 테스트 결과야: Control: 평균 전환율 3.2% (n=50000) Test: 평균 전환율 3.8% (n=50000) p-value: 0.032 신뢰도: 95% 이게 뭘 의미하는 거야? 이 테스트 결과가 실제로 의미 있는 건가? 어떻게 의사결정해야 해?" 📊 Output: - p-value 0.032의 의미 (평이한 설명) - 실제 비즈니스 임팩트 (누가 더 이득?) - 의사결정 권고사항 - 주의사항 (표본 크기 등)

💡 오늘의 인사이트 · 큰 그림

W09는 "AI를 문서화와 데이터 분석의 파트너로 쓰는" 주간입니다. MON-TUE는 기술 문서 자동화(쉬운 말 번역, API 문서, 위키 통합, 릴리스 노트), WED는 지식 그래프와 데이터 분석의 입문, THU-FRI는 실제 엑셀 데이터를 분석하고 시각화하고 정제하는 전체 데이터 파이프라인을 배웁니다.

가장 중요한 깨달음: 당신이 통계나 프로그래밍을 몰라도 괜찮습니다. AI에게 "이 데이터 뭔가 이상한데?", "이 기술 문서 평이하게 설명해줄 수 있어?"라고 물으면, AI가 전문가처럼 답합니다. 당신은 그 결과를 이해하고 의사결정하기만 하면 됩니다.

📖 주간 핵심 용어
OpenAPI / Swagger
OpenAPI Specification
REST API의 구조를 표준화된 형식으로 정의하는 방식. 이 형식으로 작성하면 자동으로 API 문서와 클라이언트 코드까지 생성할 수 있습니다.
지식 그래프
Knowledge Graph
개념(노드)과 그 관계(엣지)를 네트워크로 표현한 구조. "문서 A는 문서 B의 선행 학습이다" 같은 관계를 자동으로 발견할 수 있습니다.
크로스탭 / 피벗 테이블
Crosstab / Pivot Table
두 개 이상의 변수를 행과 열로 정리해서 다차원 데이터를 한눈에 보는 표. 예: 나이대별, 성별별 매출액을 한 표에 정렬.
p-value (유의확률)
p-value
어떤 차이가 순전히 운(무작위)으로 일어날 확률. 0.05 이하면 "통계적으로 유의미하다"고 표현. 작을수록 그 차이가 진짜라고 확신할 수 있습니다.
이상치
Outlier
데이터의 일반적인 패턴에서 크게 벗어난 값. 예: 판매량이 보통 100개인데 갑자기 1개인 달. 오류일 수도 있고, 중요한 신호일 수도 있습니다.
