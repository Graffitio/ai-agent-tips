AI에게 잘 물어보는 것이
모든 것의 시작이다

구체적으로, 그리고 하나씩 — 질문 설계의 두 가지 원칙

by Ian Jung
2026.03.11 MON
☕ 3 min read
PHASE 1 · 프롬프트 기본기
👋 오늘의 한마디

AI를 잘 쓰는 사람과 못 쓰는 사람의 차이는 기술이 아닙니다. "어떻게 질문하느냐"의 차이입니다. 오늘은 가장 기본이면서도 가장 중요한 두 가지 — 구체적으로 묻기와 하나씩 묻기를 다룹니다.

🎯 오늘의 팁 2선
#001 🌱 입문 질문 설계

구체적으로 질문하면 구체적으로 답한다

모호한 질문은 모호한 답을 만듭니다. "좋은 콘텐츠 아이디어 줘"라고 하면 AI는 일반적인 조언만 반복합니다. 하지만 산업, 타겟 오디언스, 플랫폼, 목표까지 함께 전달하면, AI는 마치 당신의 상황을 깊이 이해하고 있는 것처럼 맞춤형 답변을 만들어냅니다.

핵심: 질문에 Who, What, Why, Where, When, How의 맥락을 최대한 담으세요. 한 문장이라도 더 구체할수록 AI의 답변이 정확해집니다.

💬 모호함 vs 구체적
❌ 모호함: "블로그 글 주제 추천해줘" → 일반적인 주제 리스트만 반환 ✅ 구체적: "나는 B2B SaaS 기업의 마케팅 담당자고, 월 1,000명의 개발자를 타겟해. 우리 제품은 API 관리 플랫폼이야. 블로그를 통해 기술적 신뢰도를 높이고 싶은데, 다음 분기에 올릴 글 주제 5개를 난이도별로 제안해줘." → 당신의 사업에 딱 맞는 주제 5개

#002 🌱 입문 질문 분리

한 번에 하나씩 — 복합 질문을 쪼개라

"A 분석하고, B 작성하고, C 수정도 해줘"처럼 여러 작업을 한 번에 담으면, AI는 자원을 분산시켜 모든 것을 피상적으로 처리합니다. 깊이 있는 답변을 원한다면, 질문을 단계로 나누세요.

한 프롬프트 = 한 요청. 각 단계의 결과를 읽고 확인한 후 다음 질문을 전달하면, 각 답변의 정교함이 눈에 띄게 올라갑니다. 또한 중간에 방향을 수정할 수도 있습니다.

💬 한 번에 vs 단계별
❌ 한 번에 (피상적): "우리 경쟁사 3곳을 비교 분석해. 그리고 우리 강점도 정리하고 차별화 전략도 만들고 광고 카피도 써줘." → 4가지 모두 얕은 수준 ✅ 단계별 (깊고 구체적): [1단계] "A사, B사, C사의 마케팅 전략을 각각 분석해줘. 채널, 메시지, 타겟. [2단계] (결과 확인 후) "그 분석을 보니, 우리 제품이 가진 고유한 강점 3가지는?" [3단계] (결과 확인 후) "그 강점을 강조하는 인스타그램 광고 카피 3버전을 써줘." → 각 단계가 논리적으로 연결되고 깊음

💡 오늘의 인사이트

아무리 좋은 AI 모델도, 받는 입력이 좋아야 좋은 출력을 만듭니다. 오늘의 두 원칙 — 구체적으로, 하나씩 — 은 모든 고급 프롬프팅의 기초입니다. 내일부터는 이 기초 위에 더 많은 기법들을 쌓아갈 텐데, 이 두 가지만 지켜도 이미 대부분의 사람들보다 훨씬 나은 결과를 얻을 수 있습니다. 매번 질문하기 전에 "구체적인가? 하나인가?" 체크하는 습관을 들이세요.

📖 오늘의 용어
프롬프트
Prompt
AI에게 전달하는 모든 입력 정보를 말합니다. 질문, 지시사항, 맥락, 예시, 제약 조건 등이 포함되며, AI의 출력 품질을 결정하는 가장 중요한 요소입니다.
프롬프트 분해
Prompt Decomposition
복잡한 작업을 여러 개의 단순하고 명확한 요청으로 나누는 기법입니다. 각 단계에서 결과를 확인하며 진행하므로 정확도와 깊이가 대폭 향상됩니다.
