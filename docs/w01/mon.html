모든 것의 시작이다

구체적으로, 그리고 하나씩 — 질문 설계의 두 가지 원칙

PHASE 1 · 프롬프트 기본기
👋 오늘의 한마디

AI를 잘 쓰는 사람과 못 쓰는 사람의 차이는 기술이 아닙니다. "어떻게 질문하느냐"의 차이입니다. 오늘은 가장 기본이면서도 가장 중요한 두 가지 — 구체적으로 묻기와 하나씩 묻기를 다룹니다.

🎯 오늘의 팁 2선
#001 🌱 입문 질문 설계

구체적으로 질문하면 구체적으로 답한다

"마케팅 전략 알려줘"라고 하면 AI는 교과서 같은 일반론을 꺼냅니다. 하지만 누가, 무엇을, 왜, 어떤 조건에서 필요한지를 함께 말하면, AI는 마치 그 상황을 직접 겪은 것처럼 구체적인 답을 만들어냅니다.

핵심 공식: "[나의 상황] + [원하는 것] + [제약 조건]". 이 세 요소만 포함하면 프롬프트의 질이 확 올라갑니다.

💬 Before / After
❌ Before: "마케팅 전략 알려줘" → 4P 분석, STP 전략 등 교과서적 답변 ✅ After: "나는 반려동물 간식 D2C 브랜드를 운영해. 월 광고 예산 300만원이고, 타겟은 30대 1인 가구 반려견 보호자야. 인스타그램 중심의 3개월 마케팅 전략을 월별 액션 플랜으로 정리해줘." → 구체적인 채널·예산·일정이 포함된 답변

#002 🌱 입문 질문 분리

한 번에 하나씩 — 복합 질문을 쪼개라

"A 분석하고, B 비교하고, C 추천도 해줘"처럼 여러 작업을 한 프롬프트에 담으면 AI는 어디에 집중해야 할지 헷갈립니다. 결과적으로 모든 항목이 얕아지죠.

하나의 프롬프트 = 하나의 요청 원칙을 지키세요. 각 단계의 결과를 확인한 뒤 다음 질문으로 넘어가면, 각 답변의 깊이가 눈에 띄게 달라집니다.

💬 Before / After
❌ Before (한 번에 다 물어보기): "우리 경쟁사 3곳 분석하고, 우리 제품 장단점도 정리하고, 차별화 전략도 추천해줘. 그리고 광고 카피도 써줘." → 4가지 모두 피상적인 답변 ✅ After (하나씩 단계별로): 1️⃣ "A사, B사, C사의 핵심 강점을 각각 3가지씩 정리해줘." → 확인 후 2️⃣ "그 분석을 바탕으로, 우리 제품이 가진 차별점 3가지를 도출해줘." → 확인 후 3️⃣ "그 차별점을 살린 인스타 광고 카피를 3가지 버전으로 써줘." → 각 단계가 깊고 연결된 답변

💡 오늘의 인사이트

오늘 배운 두 가지 — "구체적으로"와 "하나씩" — 은 이후 25주 동안 배울 모든 고급 기법의 토대입니다. 어떤 기법이든 결국 "구체적으로 하나의 요청을 전달하는 것"에서 출발합니다. 오늘부터 AI에게 질문할 때 3초만 멈추고 생각해 보세요: "이 질문에 내 상황이 담겨 있는가? 요청이 하나인가?"

📖 오늘의 용어
프롬프트
Prompt
AI에게 전달하는 입력 텍스트 전체를 말합니다. 질문, 지시, 맥락, 예시 등이 모두 포함됩니다.
프롬프트 분리
Prompt Decomposition
복잡한 작업을 여러 개의 단순한 프롬프트로 나누는 기법. 각 단계에서 결과를 확인하고 다음으로 넘어가므로 정확도와 깊이가 향상됩니다.
