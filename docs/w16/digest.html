Zapier AI Actions 시작하기

Zapier AI Actions는 워크플로 중간에 AI 의사결정을 삽입하는 가장 쉬운 방법입니다. "만약 이메일 내용이 고객 불만이면 Slack에 경고를 보내라"는 식의 조건을 AI가 판단하게 하는 것이죠.

3단계 설정: (1) Trigger 정의 → (2) AI Action에서 판단 규칙 작성 → (3) 조건별 Action 연결. 프롬프트는 구체적일수록 정확합니다.

💬 실제 설정
📧 Gmail에 신규 이메일 받음 ↓ [AI Action] Prompt: "이 이메일이 긴급 고객 문의인지 판단해. 판단 기준: 사과 요청, 환불 언급, 화난 톤" ↓ 결과: True → Slack #urgent 채널에 False → 자동 분류 폴더

Make.com + AI 시나리오 설계

Make(구 Integromat)는 시각적 워크플로 빌더로, Zapier보다 복잡한 분기와 루프를 처리합니다. AI와 결합하면 배치 처리, 조건부 변환, 동적 라우팅 같은 고급 기능이 가능합니다.

핵심 팁: Router 모듈로 AI 출력값을 기반으로 여러 경로를 분기하고, Iterator로 대량 데이터를 처리할 때 각 항목마다 AI를 호출합니다.

💡 시나리오 구조
CSV 업로드 (여러 고객 데이터) ↓ [Iterator] 각 고객당 반복 실행 ↓ [AI Action] "이 고객의 위험도 점수를 0~100으로" ↓ [Router] 점수 > 70 → 전담팀 이메일 발송 점수 31~70 → 자동 이메일 + 태그 점수 < 30 → 자동 응답만 처리

Google Apps Script + AI 연동

Google Sheets에서 API 호출 없이 스크립트만으로 OpenAI나 Gemini API를 연동할 수 있습니다. 셀에 데이터가 추가되면 자동으로 AI 분석 같은 네이티브 자동화가 가능합니다.

설정: Apps Script 프로젝트 → API 활성화 → 스크립트 작성 → onEdit 또는 시간 기반 트리거. 비용이 매우 저렴한 엔터프라이즈 자동화입니다.

💻 GAS 코드 스니펫
function analyzeCustomerEmail() { const sheet = SpreadsheetApp.getSheet(); const data = sheet.getRange(2, 1, sheet.getLastRow(), 3).getValues(); data.forEach((row, idx) => { const email = row[0]; const response = callGeminiAPI(email); sheet.getRange(idx + 2, 4) .setValue(response); }); }

#154 🚀 고급

n8n으로 셀프호스팅 AI 워크플로

n8n은 오픈소스 워크플로 엔진으로, 자신의 서버에 설치하여 완전한 데이터 소유권과 커스터마이징을 갖습니다. Zapier보다 자유도가 높고, 대량 데이터 처리와 복잡한 로직에 강합니다.

주요 강점: 코드 실행 노드, HTTP 요청 노드, 데이터베이스 쿼리, 파일 시스템 접근이 모두 가능하며, 로컬 LLM(Ollama, LM Studio)도 연동 가능합니다.

🏗️ n8n 워크플로 사례
FTP 폴더 감시 ↓ 새 PDF 파일 감지 ↓ [Python Code Node] OCR로 텍스트 추출 ↓ [HTTP Request] 로컬 Ollama LLM에 분류 요청 ↓ [Conditional] 타입별 폴더에 분류 저장 ↓ [Webhook] 슬랙 알림 + 메타데이터 DB 기록

노코드 vs 코드: 선택 기준 매트릭스

"그럼 언제 노코드를 쓰고 언제 코드를 쓸까요?" — 이것이 실무 질문입니다. 규모, 속도, 유지보수성, 보안 4가지 축으로 의사결정 매트릭스를 만들었습니다.

노코드 선택: 월 데이터 < 10만 건, 변경 빈도 높음, 팀의 개발 리소스 없음, 빠른 프로토타입 필요. 코드 선택: 월 데이터 > 100만 건, 성능 중요, 복잡한 비즈니스 로직, 보안/규정 엄격.

📊 의사결정 매트릭스
프로젝트 규모별: ┌────────────┬─────────────┬──────────┐ │ 월 데이터 │ 추천 │ 이유 │ ├────────────┼─────────────┼──────────┤ │ < 1만 │ 노코드 OK │ 모든 도구│ │ 1~10만 │ 노코드 추천 │ 속도+비용│ │ 10~100만 │ 하이브리드 │ Make.com │ │ > 100만 │ 코드 추천 │ 성능 │ └────────────┴─────────────┴──────────┘

PDF 문서 분석 자동화

송장 PDF, 계약서, 영수증 같은 비정형 문서를 Zapier나 Make에서 자동 처리할 수 있습니다. Document AI나 Azure Form Recognizer 같은 서비스를 연동하면, AI가 자동으로 주요 정보(금액, 날짜, 당사자)를 추출합니다.

핵심은 매핑 단계입니다. 추출된 필드를 Google Sheets, Airtable, 또는 데이터베이스에 정규화하여 저장하세요.

📄 PDF 자동화 플로우
Gmail 첨부 PDF ↓ [Zapier: Document AI] 자동 추출: 발신인, 금액, 날짜, 내용 ↓ [AI Action (선택)] "이 문서의 긴급도를 판단" ↓ [Google Sheets] 자동 기록 + 필터링 ↓ [Conditional] 긴급 문서 → Slack 알림 일반 문서 → 아카이브

이미지 속 텍스트 인식과 분류

스크린샷, 수기 메모 사진, 명함 사진 같은 이미지 데이터를 자동으로 텍스트화하고 분류할 수 있습니다. Claude의 Vision API나 Google Vision AI를 워크플로에 삽입하면 됩니다.

실전 팁: 이미지 해상도가 낮으면 정확도가 떨어지므로, 사전에 이미지 전처리 단계를 추가하거나 사용자에게 촬영 가이드를 제공하세요.

📸 이미지 처리 워크플로
Slack에 이미지 업로드 ↓ [Zapier: Claude Vision] Prompt: "이미지에 보이는 텍스트와 카테고리(영수증/명함/문서/기타) 추출" ↓ [JSON 파싱] 텍스트: "Starbucks 5500 won..." 카테고리: "영수증" ↓ [조건부 처리] 영수증 → 경비 시스템에 기록 명함 → CRM 연락처에 추가

#158 📘 중급

이메일 첨부파일 자동 처리 워크플로

이메일의 첨부파일을 자동으로 받아서 분류, 변환, 저장하는 워크플로는 행정 업무를 획기적으로 줄입니다. Gmail/Outlook Trigger → 첨부파일 추출 → 파일 타입별 처리.

주의: 파일명이 한글이거나 특수문자를 포함하면 인코딩 오류가 발생할 수 있으므로, 파일명 정규화 단계를 추가하세요.

📎 첨부파일 자동화
Gmail에 PDF/이미지 첨부 메일 ↓ [파일명 정규화] 한글 → ASCII 변환, 띄어쓰기 제거 ↓ [파일 타입 분기] PDF 경로: → Google Drive 업로드 → OCR (Document AI) → 메타데이터 추출 → Sheets 기록 이미지 경로: → Slack #docs에 미리보기 → AI 태그 자동 추가 → 폴더별 분류 저장

대량 문서 배치 처리 설계

1,000개 이상의 문서를 한 번에 처리하려면 실시간 처리(real-time)가 아닌 배치 처리(batch)를 설계해야 합니다. 야간 또는 저부하 시간에 처리하고, 에러 문서는 별도 큐에 저장하는 방식입니다.

배치 설계 패턴: (1) 입력 폴더에서 파일 목록 생성 → (2) Iterator로 배치 사이즈(예: 50개)로 분할 → (3) 각 배치를 병렬 처리 → (4) 결과 통합 및 에러 리포팅.

🔄 배치 처리 구조
[매일 자정] ├─ Input: /inbox/*.pdf (500개) ├─ List Files ├─ [Iterator: 50개씩 배치] │ ├─ 병렬 처리 활성화 │ ├─ 각 파일 OCR │ ├─ AI 분류 (다국어 대응) │ ├─ 메타데이터 추출 │ └─ 성공 여부 기록 ├─ Error Handling │ └─ 실패 파일 → /failed 폴더 │ + #alerts 슬랙 알림 └─ Reporting └─ "처리 완료: 498/500" 실패율: 0.4% 평균 처리 시간: 12.5초/파일

#160 🚀 고급

문서 처리 파이프라인 모니터링

자동화가 늘어날수록 "뭔가 정상 작동하지 않는데 모를 수 있다"는 리스크가 높아집니다. 파이프라인 모니터링 대시보드를 만들어야 합니다: 처리율, 에러율, 평균 지연시간, 실패한 문서 목록.

구현: Zapier/Make의 로그를 Google Sheets로 내보내고, 매일 아침 자동 리포트를 이메일/Slack으로 보내기. 데이터 이상 감지 시 자동 경보를 설정합니다.

📊 모니터링 대시보드 항목
실시간 메트릭 (Google Sheet 자동 갱신): ┌─────────────────────────────┐ │ 📈 처리 통계 │ │ 전체: 2,450개 │ │ 완료: 2,413개 (98.5%) │ │ 진행중: 25개 │ │ 실패: 12개 (0.5%) │ │ 평균 처리 시간: 8.2초 │ │ │ │ ⚠️ 최근 에러 │ │ 10:45 - file_5432.pdf │ │ └ "한글 폰트 인식 실패" │ │ 09:30 - file_4891.pdf │ │ └ "스캔 해상도 낮음" │ │ │ │ 🔧 시스템 상태 │ │ API 할당량: 85% 사용 │ │ 평균 응답시간: 1.2초 │ │ 마지막 에러: 30분 전 │ └─────────────────────────────┘ ⏰ 일일 알림 (매일 09:00): "어제 처리: 347개, 성공율 99.2% 주목할 에러: 3개 (분류 참고)"

💡 이번 주의 큰 그림

이번 주 배운 5가지 핵심: (1) Zapier AI Actions로 입문 → (2) Make.com으로 중급 시나리오 설계 → (3) GAS로 Google 에코시스템 자동화 → (4) n8n으로 고급 커스터마이징 → (5) 노코드/코드 선택 기준 습득. 결론은 하나: "데이터 규모와 변경 빈도에 따라 도구를 선택하라" — 작고 자주 바뀌면 노코드, 크고 복잡하면 코드로 전환합니다. 다음 주는 이 모든 도구를 활용한 실전 케이스 스터디를 다룹니다.

📖 이번 주의 용어
노코드 플랫폼
No-Code Platform
코드를 작성하지 않고 GUI 기반으로 자동화 워크플로를 만드는 플랫폼. Zapier, Make, n8n 등이 대표. 진입 장벽이 낮고 개발 속도가 빠름.
워크플로 트리거
Workflow Trigger
자동화를 시작하는 신호. "이메일 수신", "파일 생성", "시간 도달" 등이 있음. 트리거 조건이 충족되어야 워크플로가 실행됨.
배치 처리
Batch Processing
대량의 데이터를 한 번에 모아서 주기적으로 처리하는 방식. 실시간 처리가 아니므로 서버 부하가 낮고 비용 효율적.
OCR
Optical Character Recognition
이미지나 PDF에서 텍스트를 자동으로 추출하는 기술. Document AI, Tesseract 등이 있음.
