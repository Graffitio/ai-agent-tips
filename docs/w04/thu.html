출력과 제약 걸기

보고서용 vs 코드 연동용 · 자유도를 좁혀 정밀도 올리기

PHASE 1 · 프롬프트 기본기
👋 오늘의 한마디

AI의 답변 품질은 "어떤 형식으로 답하길 원하는가"를 명시하는 순간부터 변합니다. 마크다운, JSON, 테이블, 불릿 포인트 — 각 형식은 다른 용도를 가지고 있습니다. 오늘은 구조화된 출력을 통해 정밀도를 높이는 방법을 배웁니다.

🎯 오늘의 팁 2선
#037 📈 중급 구조화 포맷

마크다운·JSON으로 출력 형식 고정하기

같은 내용이라도 형식 지시가 있으면 없을 때보다 AI의 응답이 훨씬 일관되고 정확해집니다. 마크다운은 인간이 읽기 좋은 보고서용, JSON은 코드/시스템 연동용으로 구분하면 됩니다.

핵심: 프롬프트 끝에 "다음 형식으로 답변해줘"라는 한 문장만 추가해도 AI의 출력 일관성이 크게 올라갑니다.

💬 형식 지시 예시
마크다운 (보고서·글쓰기용): "다음 결과를 마크다운 형식으로 정리해줘: ## 제목 ### 소제목 - 불릿 **굵은 글씨** 등" JSON (코드 연동용): "다음을 JSON 형식으로 반환해줘: { "key": "value", "items": [...] }"

#038 📈 중급 복합 제약

분량·톤·범위를 동시에 제약해 정밀도 올리기

프롬프트에 제약 조건을 여러 개 겹쳐서 주면, AI는 그 조건 안에서 최적의 답변을 생성합니다. 마치 "이 틀 안에서 최고의 결과를 만들어달라"는 신호죠.

3가지 제약의 조합: ① 분량(단어 수/문장 수) ② 톤(전문적/캐주얼/아이디어 형) ③ 범위(어디부터 어디까지). 이 셋을 조합하면 예측 가능하고 재사용 가능한 답변을 얻을 수 있습니다.

💬 제약 조건 결합 예시
❌ Before (제약 없음): "우리 제품 장점을 소개하는 카피 써줘" ✅ After (제약 3개 추가): "우리 제품 장점을 소개하는 카피를 써줘. [분량] 한 문단, 40~60단어 [톤] 전문적이지만 따뜻한 목소리 [범위] 기술 설명은 제외하고, 사용자 혜택만 결과: 매번 같은 수준, 같은 길이의 답변

💡 오늘의 인사이트

구조화와 제약은 "AI의 자유도를 줄인다"는 역설적인 방법으로 정밀도를 높입니다. 무한의 자유는 오히려 일관성을 해칩니다. 마크다운·JSON 형식, 단어 수 제한, 톤 지정 같은 구체적인 제약이 있을 때 AI는 그 틀 안에서 가장 최적화된 답변을 만들어냅니다. 이것이 전문가 프롬프터의 특징입니다.

📖 오늘의 용어
구조화된 출력
Structured Output
AI가 마크다운, JSON, 테이블, XML 등 미리 정의된 형식으로 답변하도록 지시하는 것. 출력 일관성과 파싱 가능성을 높입니다.
제약 겹침
Constraint Stacking
분량·톤·범위·형식 등 여러 제약을 동시에 프롬프트에 명시하는 기법. 제약이 많을수록 AI의 응답이 더 예측 가능하고 정밀해집니다.
