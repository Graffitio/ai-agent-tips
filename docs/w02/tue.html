#013 🌱 입문 파일 활용

파일 업로드로 맥락 제공하기

"우리 회사 소개를 기반으로..."이라고 말로 설명하는 것보다, PDF 파일을 직접 업로드하는 것이 훨씬 정확합니다. AI가 실제 텍스트를 읽고 분석하니까요.

파일 업로드는 텍스트 문서(PDF, DOCX), 이미지(PNG, JPG), 스프레드시트(XLSX) 등을 지원합니다. 특히 이미지 내 텍스트, 차트, 디자인까지 인식하므로 스크린샷도 활용 가치가 높습니다.

📎 활용 사례
✅ 파일 업로드 활용 시나리오: 1. 계약서/약관 분석 → PDF 업로드 + "이 계약서의 위험 요소를 찾아줘" 2. 디자인 검수 → 배너 이미지 업로드 + "이 배너의 개선점을 3가지 지적해줘" 3. 데이터 정리 → 엑셀 파일 업로드 + "이 데이터를 월별로 분석해줘" 4. 문서 요약 → 장문의 보고서 PDF + "핵심 내용 3줄로 정리해줘"

#014 🌱 입문 히스토리 관리

대화 기록 관리와 정리

AI와의 대화를 계속 이어가면 맥락이 쌓이지만, 동시에 정보가 복잡해집니다. 언제 새 대화를 시작하고, 언제 이전 대화를 이어갈지 판단하는 것이 중요합니다.

긴 대화 전략은 이전 답변을 기반으로 반복 개선할 때 좋고, 새 대화 전략은 완전히 다른 주제나 깔끔한 시작이 필요할 때 효과적입니다. 핵심은 맥락의 "신선도"를 관리하는 것입니다.

💭 Before / After
❌ Bad: 한 대화에서 100개 메시지 "마케팅 전략 내줘" → "이제 영업 스크립트 만들어줘" → "회사 로고 리브랜딩해줘" → "인사 정책 검토해줘" (완전히 무관한 주제들) → AI의 맥락이 혼란스러워짐 ✅ Good: 주제별 대화 분리 📌 대화 1: 마케팅 전략 (진행 중 개선) 📌 대화 2: 영업 전략 (새로 시작) 📌 대화 3: 브랜딩 (새로 시작) 📌 대화 4: 인사 정책 (새로 시작) 각 대화는 3-5개 메시지 정도에서 결론을 내리고 정리하기

💡 오늘의 인사이트

파일 업로드 + 대화 관리는 "구체적이고 하나씩"의 다음 단계입니다. AI에게 정보를 정확하게 주고, 대화를 깔끔하게 유지하면 — 신뢰도 높은 답변이 나옵니다. 다음주부터는 이 기초 위에서 고급 프롬프트 기법들을 배워갑니다. 지금 이 두 가지를 완벽히 다루면, 나머지는 그 위에 쌓이는 것입니다.

📖 오늘의 용어
파일 업로드
File Upload
AI 챗봇에 문서, 이미지, 스프레드시트 등의 파일을 직접 제공하는 기능. 입력으로 텍스트만 사용하는 것보다 정확도와 깊이가 높아집니다.
대화 맥락
Conversation Context
AI가 이전 메시지들을 기억하고 참고하는 상태. 맥락이 정확할수록 답변이 일관성 있지만, 대화가 길어질수록 맥락이 흐려질 수 있습니다.
